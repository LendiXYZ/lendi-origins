# Decisions — 2026-04-10

## DEC-001: Bootstrap product from platform-modules

**Decision:** Use platform-modules as scaffold for Lendi product app + backend.

**Rationale:** Faster path to Wave 2 than building from scratch. Gets ZeroDev, SIWE/JWT, Clean
Architecture, and escrow patterns out of the box.

**Trade-offs:** Inherits some patterns that may not perfectly fit Lendi's domain (e.g. escrow-centric
entities vs income-centric). Will need to customize entity layer.

## DEC-002: Four core entities for Wave 2

**Decision:** Worker, Lender, IncomeEvent, Loan as initial backend entities.

**Rationale:** Maps directly to the InformalProof contract model. Worker/Lender mirror on-chain
registration. IncomeEvent is an off-chain index (no amounts). Loan wraps the escrow flow.

**Trade-offs:** Lean entity set — may need UserSession entity later for auth bridge.
